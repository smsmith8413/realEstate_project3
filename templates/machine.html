Machine Learning with Pycaret

After importing the data I quickly ran a df.info() to print a concise summary of the DataFrame to understand what information we are working with. Including the index dtype and column dtypes, non-null values and memory usage.

The PyCaret setup is the first and main step to do any kind of machine learning experiment. The target is set equal to “ClosePrice” since that is the parameter we are trying to predict.

The “compare_models()” on the caret package website lists variations of predictive analytics models that are available within the caret framework. It starts with the best model then gradually goes down in quality of what the best model for the data would be. All these models may be prepared, tuned, fit and evaluated with a common set of caret functions.

We “plot_model()” for the top few analytic models the “compare_models()” suggested and decided to go with Light GBM.

In this visual the select features in the dataset that contributes the most in predicting the target variable “ClosePrice” for the LightGBM model are shown. Working with selected features instead of all the features reduces the risk of over-fitting, improves accuracy, and decreases the training time.
